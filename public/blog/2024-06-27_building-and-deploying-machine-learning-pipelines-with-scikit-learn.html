Building and Deploying Machine Learning Pipelines with scikit-learn

In this article, we will walk through the process of building and deploying machine learning pipelines using the Pipeline class from scikit-learn. We will use a dataset from the Titanic competition to illustrate the process.

Introduction to Pipelines

A machine learning pipeline in scikit-learn is a way to streamline a series of data processing and modeling steps. Pipelines help ensure that the same transformations are applied during both training and testing, preventing data leakage and making your workflow cleaner and more reproducible.

Dataset Overview

We will use the Titanic dataset, which contains information about passengers and whether they survived the Titanic disaster. The goal is to build a model that predicts survival based on passenger attributes.

Step-by-Step Guide

Step 1: Load and Explore the Data

import pandas as pd

# Load the dataset
df = pd.read_csv('train.csv')
print(df.head())

Step 2: Drop Irrelevant Columns

We drop columns that won’t be useful for prediction.

df.drop(columns=['PassengerId', 'Name', 'Ticket', 'Cabin'], inplace=True)

Step 3: Split the Data

Split the data into training and testing sets.

from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    df.drop(columns=['Survived']), 
    df['Survived'], 
    test_size=0.2, 
    random_state=42
)

Step 4: Create Transformation Pipelines

Imputation Transformer

Handle missing values.

from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer

trf1 = ColumnTransformer([
    ('impute_age', SimpleImputer(), [2]),  # Impute Age
    ('impute_embarked', SimpleImputer(strategy='most_frequent'), [6])  # Impute Embarked
], remainder='passthrough')

One-Hot Encoding

Convert categorical variables into numeric.

from sklearn.preprocessing import OneHotEncoder

trf2 = ColumnTransformer([
    ('ohe_sex_embarked', OneHotEncoder(sparse=False, handle_unknown='ignore'), [1, 6])  # One-Hot Encode Sex and Embarked
], remainder='passthrough')

Scaling

Scale the features to a given range.

from sklearn.preprocessing import MinMaxScaler

trf3 = ColumnTransformer([
    ('scale', MinMaxScaler(), slice(0, 10))  # Scale all features
])

Feature Selection

Select the most important features.

from sklearn.feature_selection import SelectKBest, chi2

trf4 = SelectKBest(score_func=chi2, k=8)

Step 5: Create the Model

Use a decision tree classifier.

from sklearn.tree import DecisionTreeClassifier

trf5 = DecisionTreeClassifier()

Step 6: Create and Train the Pipeline

Combine all transformations and the model into a single pipeline.

from sklearn.pipeline import Pipeline

pipe = Pipeline([
    ('trf1', trf1),
    ('trf2', trf2),
    ('trf3', trf3),
    ('trf4', trf4),
    ('trf5', trf5)
])

# Train the pipeline
pipe.fit(X_train, y_train)

Step 7: Evaluate the Pipeline

Evaluate the model on the test data.

from sklearn.metrics import accuracy_score

y_pred = pipe.predict(X_test)
print(accuracy_score(y_test, y_pred))

Step 8: Cross-Validation

Use cross-validation to check the model’s robustness.

from sklearn.model_selection import cross_val_score

print(cross_val_score(pipe, X_train, y_train, cv=5, scoring='accuracy').mean())

Step 9: Hyperparameter Tuning

Use grid search to find the best hyperparameters.

from sklearn.model_selection import GridSearchCV

params = {
    'trf5__max_depth': [1, 2, 3, 4, 5, None]
}

grid = GridSearchCV(pipe, params, cv=5, scoring='accuracy')
grid.fit(X_train, y_train)
print(grid.best_score_)
print(grid.best_params_)

Step 10: Save and Load the Pipeline

Export the trained pipeline to a file for later use.

import pickle

pickle.dump(pipe, open('pipe.pkl', 'wb'))

Load the pipeline and use it for predictions.

pipe = pickle.load(open('pipe.pkl', 'rb'))

# Example user input
test_input = np.array([2, 'male', 31.0, 0, 0, 10.5, 'S'], dtype=object).reshape(1, 7)
print(pipe.predict(test_input))

Conclusion

Pipelines in scikit-learn provide a powerful way to manage the entire machine learning workflow, from preprocessing to model training and evaluation. By following this guide, you can build robust and reproducible pipelines for your own machine learning projects.
