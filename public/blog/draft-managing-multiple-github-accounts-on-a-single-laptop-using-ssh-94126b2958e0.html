Managing Multiple GitHub Accounts on a Single Laptop Using SSH

Working with multiple GitHub accounts on the same machine can be tricky, especially when each account has its own repositories. SSH keys make it simple to manage this setup without logging in and out repeatedly. Here’s a step-by-step guide to configure two GitHub accounts on a single laptop.

Step 1: Install Git and Prepare the SSH Folder

Make sure Git is installed on your laptop. Then create a .ssh folder to store your keys:

mkdir -p ~/.ssh

Step 2: Generate the First SSH Key for Your Personal Account

Create an SSH key using the ed25519 algorithm. Save it as the default id_ed25519 and optionally add a passphrase:

ssh-keygen -t ed25519 -C "personal@example.com"

Step 3: Start the SSH Agent and Add Your Key

Start the SSH agent and add your personal key. Use quotes if your username has spaces:

eval $(ssh-agent -s)
ssh-add "/c/Users/Tech Mehal/.ssh/id_ed25519"

Step 4: Add Your First Key to GitHub

Copy the public key and add it to your personal GitHub account:

cat "/c/Users/PC Name/.ssh/id_ed25519.pub"

Go to GitHub → Settings → SSH and GPG keys → New SSH key, give it a descriptive title like Laptop - Personal, and paste the key.

Step 5: Test the Connection

Verify your connection:

ssh -T git@github.com

GitHub should respond with a success message.

Step 6: Generate a Second SSH Key for Your Work Account

Generate a second SSH key and save it as id_ed25519_second to avoid overwriting the first:

ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_second

Add a passphrase if desired.

Step 7: Add the Second Key to the SSH Agent

Add your work key to the SSH agent:

ssh-add "/c/Users/PC Name/.ssh/id_ed25519_second"

Step 8: Add the Second Key to Your Work GitHub Account

Copy the public key and add it to your second GitHub account:

cat ~/.ssh/id_ed25519_second.pub

Go to Settings → SSH and GPG keys → New SSH key, give it a title like Laptop - Work Account, and paste the key.

Step 9: Configure SSH to Handle Both Accounts

Create an SSH config file to manage which key is used for each account:

nano ~/.ssh/config

Add the following:

# Personal GitHub account
Host github.com-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519

# Work GitHub account
Host github.com-work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_second

Step 10: Test Both Connections

Verify that both accounts are recognized:

ssh -T git@github.com-personal
ssh -T git@github.com-work

Both should return:

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

Step 11: Use the Correct Account for Repositories

Clone repositories using the correct host:

For personal repos:

git clone git@github.com-personal:username/repo.git

For work repos:

git clone git@github.com-work:username/repo.git

To update an existing repository:

git remote set-url origin git@github.com-work:username/repo.git

Outcome

Your laptop now has two GitHub accounts fully set up with SSH keys. Each repository uses its corresponding account without conflicts, and the SSH configuration automatically handles which key to use.
