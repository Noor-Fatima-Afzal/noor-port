🌳 Reconstructing a Binary Tree from Traversals: Preorder + Inorder, Postorder + Inorder

Have you ever wondered how a binary tree can be rebuilt just from how it’s traversed? It might sound like magic, but with the right pair of traversal sequences and a few simple rules, we can fully reconstruct a binary tree step by step.

In this article, we’ll explore:

  • What traversals are
  • Why some combinations work for reconstruction
  • How to rebuild a binary tree using inorder + preorder or inorder + postorder
  • Visual examples to make it click

Let’s dig in! 🌱

🔁 What Are Tree Traversals?

A tree traversal is the process of visiting each node in a binary tree in a specific order. The three most common types are:

  1. Inorder (Left → Root → Right)
  2. Preorder (Root → Left → Right)
  3. Postorder (Left → Right → Root)

Each one gives a different perspective on the structure of the tree. But when used in pairs — especially with inorder — we can reverse-engineer the original tree.

Just remember these two golden rules:

🟡 In preorder, the first node is the root.
 🔵 In postorder, the last node is the root.

🔷 1. Reconstruct Using Inorder + Preorder

✅ Given:

  • Preorder (Root → Left → Right):
     a b d g h e i c f j
  • Inorder (Left → Root → Right):
     g d h b e i a f j c

🧩 Step-by-Step:

Step 1: Root = a (first of preorder)

  • Inorder: g d h b e i → Left Subtree
     f j c → Right Subtree

→ Root: a

      a
     / \
 [left] [right]

Step 2: Left Subtree Root = b (next in preorder)

  • Inorder (for left): g d h b e i
  • Root = b
  • Inorder left of b: g d h → Left
  • Inorder right of b: e i → Right

→ Add b as left child of a

      a
     / \
    b   ?
   / \
[?]  [?]

Step 3: Left Subtree of b → Root = d

  • Inorder = g d h
  • Root = d
  • Left: g
  • Right: h

→ Add d as left of b, with g (left) and h (right)

        a
       / \
      b   ?
     / \
    d   e
   / \   \
  g   h   i

Step 4: Right Subtree of b → Root = e

  • Inorder = e i
  • Root = e
  • Right child = i

→ Add e as right child of b, i as its right child.

Step 5: Now handle Right Subtree of a → Root = c

  • Inorder = f j c
  • Root = c
  • Left = f j

→ Add c as right child of a

Step 6: Left Subtree of c → Root = f, Right = j

→ f as left child, j as right of f

        a
       / \
      b   c
     / \  /
    d   e f
   / \    \
  g   h    j
       \
        i

✅ Final Tree (Inorder + Preorder):

          a
        /   \
       b     c
      / \   /
     d   e f
    / \   \  \
   g   h   i  j

🔷 2. Reconstruct Using Inorder + Postorder

✅ Given:

  • Postorder (Left → Right → Root):
     g h d i e b j f c a
  • Inorder:
     g d h b e i a f j c

🧩 Step-by-Step:

Step 1: Root = a (last in postorder)

  • Inorder: Left of a = g d h b e i, Right = f j c

→ Root = a

      a
     / \
[Left] [Right]

Step 2: Right Subtree Root = c (from postorder, before a)

  • Inorder = f j c
  • Root = c
  • Left of c = f j

→ Add c as right of a

Step 3: f becomes left of c, and j as right of f

→ From postorder: j f

      a
     / \
    ?   c
       /
      f
       \
        j

Step 4: Now handle Left Subtree of a → Root = b (next back in postorder)

  • Inorder = g d h b e i
  • Root = b
  • Left of b = g d h
  • Right of b = e i

→ Add b as left child of a

Step 5: Left of b → Root = d, with g (left) and h (right)

→ From postorder: g h d

Step 6: Right of b → Root = e, with i as right child

→ From postorder: i e

✅ Final Tree (Inorder + Postorder):

          a
        /   \
       b     c
      / \   /
     d   e f
    / \   \  \
   g   h   i  j

🧠 Conclusion

As you can see, both traversal pairs lead to the same reconstructed tree:

          a
        /   \
       b     c
      / \   /
     d   e f
    / \   \  \
   g   h   i  j
  • Preorder helps you build top-down (starting from root).
  • Postorder helps you build bottom-up (starting from leaves).

🧠 Final Thoughts

The ability to reconstruct a binary tree comes down to knowing where the root is, and then dividing the tree based on inorder’s layout. Whether it’s:

  • Start from the beginning (preorder), or
  • Start from the end (postorder),

Combine that with inorder to split the tree, and you’re good to go!
