Hashing in Data Structures and Algorithms (DSA)

Hashing is a very important concept in Data Structures and Algorithms (DSA). It helps us store and retrieve data quickly — in nearly constant time O(1) on average!
 Let’s break down hashing in a very easy way.

What is Hashing?

Hashing means converting data into a fixed-size value (usually an integer) by using a hash function.
 This hash value tells us where to store or find the data.

Think of it like assigning a locker number to every student’s bag.

  • Instead of checking all bags, you directly go to the locker number.

Important Terms in Hashing

1. Hash Function

  • A hash function takes an input (like a name or number) and returns an integer (the index).
  • It decides where in memory the data will be placed.
  • Good hash functions distribute data evenly to avoid collisions.

Example:
 hash("apple") → 3
 hash("banana") → 7

2. Bucket Array

  • A bucket array is like a big list (or array) where each slot can hold data.
  • The index to place the data is decided by the hash function.
  • It’s called bucket because each index may store one or more values if needed.

HashSet and HashMap

These are two common uses of hashing:

HashSet

  • Stores only values.
  • No duplicates allowed.
  • Example: {1, 4, 6, 9}

In Python: set(),
 In Java: HashSet<>

HashMap

  • Stores key-value pairs.
  • Each key must be unique.
  • Example: {1: "Apple", 2: "Banana"}

In Python: dict(),
 In Java: HashMap<>

Collision in Hashing

Collision happens when two inputs get the same hash index.
 This is common because the number of inputs may be more than available slots.

Example:

  • hash("cat") → 5
  • hash("dog") → 5

Both cat and dog want the same locker!

Load Factor

  • Load Factor = (Number of entries) / (Number of buckets)
  • It measures how full the bucket array is.

Example:

  • 8 entries, 10 buckets → Load Factor = 0.8

When load factor becomes too high (e.g., 0.7 or more), we resize (expand) the array to avoid too many collisions.

Collision Handling Techniques

Collisions are common — so we need ways to handle them.

There are two main techniques:

1. Separate Chaining (Open Hashing)

  • Instead of storing only one value per index, we store a linked list (or chain) of values.
  • All elements that hash to the same index are linked together.

Example:

IndexData5[cat → dog → cow]

Advantage:

  • Easy to implement.
  • Only grows chain length at one index.

2. Open Addressing (Closed Hashing)

  • If a bucket is full, we search for another empty bucket.
  • All elements stay inside the bucket array — no linked list.

There are different ways to search for a new bucket. These are called probing techniques:

Probing Methods

a. Linear Probing

  • If the desired bucket is full, try the next one (index + 1).
  • If that’s also full, move to (index + 2), and so on.

Example:
 Suppose hash("apple") = 5 but 5 is full → check 6, then 7, etc.

Problem:

  • Clustering: lots of items may form groups together.

b. Quadratic Probing

  • Instead of checking linearly, we check using squares.

Next indexes:
 (index + 1²), (index + 2²), (index + 3²), etc.

Example:
 If 5 is full, try 6 (5+1), 9 (5+4), 14 (5+9)…

Advantage:

  • Reduces clustering compared to linear probing.

c. Double Hashing

  • We use a second hash function to decide the step size for the next probe.

Example:
 hash2(key) = 7

If bucket is full at index 5, next index = (5 + 7) % array size.

Advantage:

  • Very good at spreading entries.
  • Less clustering.

Final Words

Hashing makes storing and searching data super-fast.
 But we must handle collisions properly using chaining or probing methods.
 Choosing a good hash function and keeping load factor low are keys to efficient hashing.

Hashing is used everywhere — from databases to caches to sets and dictionaries in programming!
