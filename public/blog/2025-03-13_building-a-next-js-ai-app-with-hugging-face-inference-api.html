Building a Next.js AI App with Hugging Face Inference API

Artificial Intelligence (AI) is revolutionizing the way we build applications, making them smarter and more interactive. In this guide, we will build a Next.js application that integrates with Hugging Face’s Inference API to perform AI-driven tasks, such as text generation.

By the end of this tutorial, you’ll have a fully functional AI-powered Next.js app, ready for deployment.

Step 1: Setting Up the Next.js Project

First, let’s create a new Next.js application. Open your terminal and run the following command:

npx create-next-app@latest huggingface-next-app
cd huggingface-next-app

When prompted with setup options, select:

  • Would you like to use TypeScript? ❌ No
  • Would you like to use ESLint? ✅ Yes
  • Would you like to use Tailwind CSS? ❌ No (Optional: You can add it later)
  • Would you like to use src/ directory? ✅ Yes (This is important)
  • Would you like to use experimental app/ directory? ❌ No (We are using pages/ structure)
  • Would you like to use a import alias (@/*)? ✅ Yes

Step 2: Installing Dependencies

We need to install Hugging Face’s Inference API library to interact with AI models:

npm install @huggingface/inference

We will also install Axios for making API requests:

npm install axios

Step 3: Setting Up the Hugging Face Inference API

Create a new file inside the project directory:

huggingface-next-app/
│── src/
│   ├── pages/
│   │   ├── index.js      <-- Main Page (UI)
│   │   ├── api/
│   │   │   ├── inference.js  <-- API Route
│── package.json
│── next.config.js
│── .gitignore
│── README.md
│── .env.local  <-- Secure API Key

Inside src/pages/api/inference.js, paste the following code:

import { InferenceClient } from "@huggingface/inference";

const HF_TOKEN = process.env.HF_TOKEN;  // Store token securely in environment variables
const inference = new InferenceClient(HF_TOKEN);

export default async function handler(req, res) {
    if (req.method !== "POST") {
        return res.status(405).json({ error: "Method not allowed" });
    }

    try {
        const { text } = req.body;

        if (!text || typeof text !== "string") {
            return res.status(400).json({ error: "Invalid request. 'text' field is required." });
        }

        const response = await inference.chatCompletion({
            model: "google/gemma-2b-it",
            messages: [{ role: "user", content: text }],
            max_tokens: 512,
        });

        if (!response || !response.choices || response.choices.length === 0) {
            throw new Error("Invalid response from Hugging Face API.");
        }

        res.status(200).json({ response: response.choices[0].message });
    } catch (error) {
        console.error("Error calling Hugging Face API:", error);

        res.status(500).json({
            error: "Error in AI response",
            details: error.message || "Unknown error",
        });
    }
}

Breaking Down the API Code

  1. Imports Hugging Face’s API Client for text generation.
  2. Reads user input from the request body and sends it to the model.
  3. Returns the generated AI response to the frontend.

Step 4: Building the Frontend UI

Inside src/pages/index.js, replace the existing code with:

import { useState } from "react";
import axios from "axios";

export default function Home() {
    const [userInput, setUserInput] = useState("");
    const [aiResponse, setAiResponse] = useState("");
    const handleSubmit = async (event) => {
        event.preventDefault();
        setAiResponse("Generating response...");
        try {
            const res = await axios.post("/api/inference", { text: userInput });
            setAiResponse(res.data.response);
        } catch (error) {
            setAiResponse("Error generating response. Try again!");
        }
    };
    return (
        <div className="container">
            <h1>Hugging Face AI Chat</h1>
            <form onSubmit={handleSubmit}>
                <input 
                    type="text" 
                    placeholder="Ask me anything..." 
                    value={userInput} 
                    onChange={(e) => setUserInput(e.target.value)} 
                    required 
                />
                <button type="submit">Generate</button>
            </form>
            <p><strong>Response:</strong> {aiResponse}</p>
            <style jsx>{`
                .container {
                    text-align: center;
                    max-width: 600px;
                    margin: auto;
                    padding: 50px;
                }
                input {
                    width: 80%;
                    padding: 10px;
                    margin: 10px;
                }
                button {
                    padding: 10px;
                    background: #0070f3;
                    color: white;
                    border: none;
                    cursor: pointer;
                }
                button:hover {
                    background: #005bb5;
                }
            `}</style>
        </div>
    );
}

Step 5: Running the Next.js App

Run the development server with:

npm run dev

Now, visit http://localhost:3000/ in your browser. You should see an input field where you can type a message and receive an AI-generated response.

Step 6: Storing API Key Securely

We never want to hardcode the Hugging Face API token. Instead, create a .env.local file in the root directory:

HF_TOKEN=your_huggingface_token_here

Then, update next.config.js to load environment variables:

module.exports = {
    env: {
        HF_TOKEN: process.env.HF_TOKEN,
    },
};

Restart the server for changes to take effect:

npm run dev

Step 7: Deploying the App on Vercel

  1. Push the project to GitHub
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/huggingface-next-app.git
git push -u origin main

Deploy to Vercel

Install Vercel CLI:

  • npm install -g vercel

Deploy the app:

  • vercel

Set environment variables in Vercel:

Go to Vercel Dashboard

  • Select your project → Settings → Environment Variables
  • Add HF_TOKEN

Run the deployed app

  • Vercel provides a live URL for your app! 🎉
