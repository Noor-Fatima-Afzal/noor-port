Save Your Work Without Committing by Mastering Git Stash

Every developer has faced it: you’re mid-way through a feature, and suddenly a critical bug needs attention. You don’t want to commit unfinished code, but you also don’t want to lose your progress. Enter git stash — your temporary storage for uncommitted changes.

Let’s dive into how to use it effectively.

Create a Practice Repository

Before we experiment, let’s set up a test project:

mkdir git-stash-practice
cd git-stash-practice
git init

Create a file:

echo "Hello World" > app.txt

Add and commit it:

git add .
git commit -m "Initial commit"

Now you have a clean repository to play with.

Make Changes (Uncommitted Work)

Edit the file to simulate work in progress:

echo "New feature line" >> app.txt

Check the status:

git status

You’ll see:

modified: app.txt

This indicates you have changes that aren’t committed yet.

Use git stash

To temporarily save these changes:

git stash

Git stores your changes and cleans your working directory. Verify:

git status

Output:

nothing to commit, working tree clean

Your edits are safely hidden in the stash.

View Stashed Changes

To see all stashed changes:

git stash list

Example:

stash@{0}: WIP on main: Initial commit

You now have one stash stored.

Restore Stashed Changes

Bring your changes back into the working directory:

git stash apply

Check the status:

git status

Your modifications are restored, ready for further editing.

Remove the Stash After Applying

apply keeps the stash. To restore and remove it:

git stash pop

This applies the stash and deletes it from the list.

Create Named Stash (Best Practice)

Instead of generic messages, name your stash for clarity:

git stash push -m "login feature changes"

git stash list

Now you can easily identify the stash you need.

Apply a Specific Stash

If multiple stashes exist, restore a specific one:

git stash apply stash@{0}

This ensures you’re applying the right set of changes.

Delete a Stash

Remove one stash:

git stash drop stash@{0}

Remove all stashes:

git stash clear

Keep your stash list tidy to avoid confusion.

Real-World Scenario: Why Developers Use Stash

Imagine working on a feature branch, and a critical bug pops up. You can’t commit unfinished code. Here’s how stash saves the day:

git stash
git checkout bug-fix-branch
# Fix the bug
git checkout feature-branch
git stash pop

Your unfinished work returns, and the bug is resolved — all without polluting your commit history.

Conclusion

git stash is a lifesaver for developers juggling multiple tasks. It provides a safe, temporary place for work in progress, letting you switch contexts without committing half-done features. By using named stashes and cleaning up regularly, you can keep your workflow clean and efficient.
