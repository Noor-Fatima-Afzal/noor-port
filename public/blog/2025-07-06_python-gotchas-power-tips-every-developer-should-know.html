Python Gotchas & Power Tips Every Developer Should Know

Python is beloved for its simplicity and elegance, but that elegance can sometimes hide subtle behavior that surprises even experienced developers. In this article, we’ll unpack some crucial Python patterns and common pitfalls — complete with code and output — to help you write cleaner, more predictable code.

List Mutation: += vs +

Let’s start with list mutation, which is often misunderstood:

a = [1, 2, 3]
b = a
a += [4]  # modifies a in-place
print(a, b)  # [1, 2, 3, 4] [1, 2, 3, 4]

But…

a = [1, 2, 3]
b = a
a = a + [4]  # creates a new list
print(a, b)  # [1, 2, 3, 4] [1, 2, 3]

✅ Key takeaway: += modifies the object in place. + creates a new object.

.append() vs .extend()

lst = [1, 2]
lst.append([3, 4])
print(lst)  # [1, 2, [3, 4]]

lst = [1, 2]
lst.extend([3, 4])
print(lst)  # [1, 2, 3, 4]

Use .append() to add a single element. Use .extend() to add multiple.

Identity vs Equality: is vs ==

a = 256
b = 256
print(a is b)  # True (Python caches small integers)

x = 1000
y = 1000
print(x is y)  # False (no caching beyond 256)

Use == for value equality. Use is for object identity.

Shallow Copy vs Deep Copy

import copy

a = [[1, 2], [3, 4]]
b = a[:]  # shallow copy
a[0][0] = 999
print(b)  # [[999, 2], [3, 4]]
a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)
a[0][0] = 999
print(b)  # [[1, 2], [3, 4]]

Shallow copies copy references. Deep copies copy nested objects.

Dangerous Defaults

def add_item(item, lst=[]):
    lst.append(item)
    return lst

print(add_item(1))  # [1]
print(add_item(2))  # [1, 2]

Avoid mutable default arguments! Use:

def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

Consuming Generators

gen = (x for x in range(5))
print(2 in gen)  # True
print(2 in gen)  # False (already consumed!)

Generators are exhaustible. Wrap them with list() if needed multiple times.

Lambda Closures in Loops

funcs = [lambda: i for i in range(5)]
print([f() for f in funcs])  # [4, 4, 4, 4, 4]

Each lambda refers to the same i. Fix using:

funcs = [lambda i=i: i for i in range(5)]
print([f() for f in funcs])  # [0, 1, 2, 3, 4]

LEGB Scope & global / nonlocal

x = 5

def outer():
    x = 10
    def inner():
        nonlocal x
        x += 1
    inner()
    return x
print(outer())  # 11

nonlocal affects enclosing scopes. global affects module-level scope.

Mutable Contents Inside Immutable Tuples

t = ([1, 2],)
t[0].append(3)
print(t)  # ([1, 2, 3],)

Tuples are immutable, but their contents may not be.

List Comprehensions vs Generators

import time

t0 = time.time()
sum([i for i in range(10000000)])
print("List time:", time.time() - t0)
t0 = time.time()
sum(i for i in range(10000000))
print("Generator time:", time.time() - t0)

Generators are more memory efficient, but not always faster.

finally Blocks Always Run

try:
    raise ValueError("oops")
finally:
    print("Finally block runs!")  # Always executed

Even if you return or raise, the finally block will run.

Lazy Evaluation Caveat

gen = (x for x in range(10) if 1/0 == 0)
# Error is raised immediately, not lazily!

The filter clause in generator expressions is evaluated during creation.

*args and **kwargs

def greet(*args, **kwargs):
    print(args, kwargs)

greet("Hi", name="Alice")  # ('Hi',) {'name': 'Alice'}

Use *args for positional and **kwargs for keyword arguments.

Metaclasses: __new__ vs __init__

class MyClass:
    def __new__(cls):
        print("Creating instance")
        return super().__new__(cls)

def __init__(self):
        print("Initializing instance")
obj = MyClass()

__new__ creates the object, __init__ initializes it.

Conclusion

Python’s elegance comes with a need to understand its underlying behaviors. From mutable defaults to deep copying, closures to metaclasses, knowing these patterns makes your code more robust and maintainable.
