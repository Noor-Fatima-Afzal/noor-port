Building Your First Convolutional Neural Network with PyTorch: A Beginner’s Guide

Convolutional Neural Networks (CNNs) have revolutionized computer vision, powering everything from face recognition to self-driving cars. If you’ve been curious about deep learning but felt intimidated by the complexity, you’re in the right place. In this tutorial, we’ll build a CNN from scratch using PyTorch, one of the most popular deep learning frameworks today.

What is a Convolutional Neural Network?

Before diving into code, let’s understand what makes CNNs special. Unlike traditional neural networks that treat all pixels equally, CNNs are designed to recognize spatial patterns in images. They work by applying small filters across an image, detecting features like edges, textures, and eventually complex objects.

Think of it like looking at a photograph through a magnifying glass, examining small sections at a time. The network learns which patterns are important for identifying what’s in the image.

Why PyTorch?

PyTorch has become a favorite among researchers and practitioners for its intuitive design and flexibility. It feels more like writing regular Python code compared to other frameworks, making it perfect for beginners. Plus, it has excellent documentation and a supportive community.

Understanding the Dataset

We’ll use the CIFAR-10 dataset, which contains 60,000 tiny color images (32x32 pixels) across 10 categories: airplanes, cars, birds, cats, deer, dogs, frogs, horses, ships, and trucks. It’s perfect for learning because it’s small enough to train quickly but complex enough to be interesting.

Building the CNN Architecture

Let’s create our CNN step by step. I’ll explain each component as we go:

import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self):
        super(SimpleCNN, self).__init__()
        
        # First convolutional layer
        # Input: 3 channels (RGB), Output: 32 channels, Kernel size: 3x3
        self.conv1 = nn.Conv2d(3, 32, 3, padding=1)
        
        # Second convolutional layer
        self.conv2 = nn.Conv2d(32, 64, 3, padding=1)
        
        # Third convolutional layer
        self.conv3 = nn.Conv2d(64, 128, 3, padding=1)
        
        # Max pooling layer
        self.pool = nn.MaxPool2d(2, 2)
        
        # Fully connected layers
        self.fc1 = nn.Linear(128 * 4 * 4, 512)
        self.fc2 = nn.Linear(512, 10)
        
        # Dropout for regularization
        self.dropout = nn.Dropout(0.25)
    
    def forward(self, x):
        # Convolutional layers with ReLU activation and pooling
        x = self.pool(F.relu(self.conv1(x)))  # 32x32 -> 16x16
        x = self.pool(F.relu(self.conv2(x)))  # 16x16 -> 8x8
        x = self.pool(F.relu(self.conv3(x)))  # 8x8 -> 4x4
        
        # Flatten for fully connected layer
        x = x.view(-1, 128 * 4 * 4)
        
        # Fully connected layers
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.fc2(x)
        
        return x

Breaking Down the Architecture

Convolutional Layers: These are the heart of our CNN. Each Conv2d layer applies filters that learn to detect features. The first layer might learn edges, the second layer combines edges into shapes, and the third learns more complex patterns.

Pooling Layers: MaxPool2d reduces the spatial dimensions by taking the maximum value in each 2x2 region. This makes our network more efficient and helps it focus on the most important features.

Fully Connected Layers: After extracting features with convolutions, we flatten everything and pass it through traditional neural network layers to make our final classification.

Dropout: This randomly turns off some neurons during training, preventing the network from relying too heavily on any single feature. It’s like teaching a student not to memorize but to truly understand.

Loading and Preparing Data

Now let’s load our dataset and prepare it for training:

import torchvision
import torchvision.transforms as transforms
from torch.utils.data import DataLoader

# Data augmentation and normalization
transform_train = transforms.Compose([
    transforms.RandomHorizontalFlip(),
    transforms.RandomCrop(32, padding=4),
    transforms.ToTensor(),
    transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
])
transform_test = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
])
# Load CIFAR-10 dataset
trainset = torchvision.datasets.CIFAR10(
    root='./data', 
    train=True,
    download=True, 
    transform=transform_train
)
testset = torchvision.datasets.CIFAR10(
    root='./data', 
    train=False,
    download=True, 
    transform=transform_test
)
# Create data loaders
trainloader = DataLoader(trainset, batch_size=64, shuffle=True, num_workers=2)
testloader = DataLoader(testset, batch_size=64, shuffle=False, num_workers=2)
# Class names for reference
classes = ('plane', 'car', 'bird', 'cat', 'deer', 
           'dog', 'frog', 'horse', 'ship', 'truck')

Data Augmentation: We apply random flips and crops to training images. This artificially expands our dataset and helps the model generalize better, like showing a child photos of dogs from different angles.

Normalization: We scale pixel values to a standard range, which helps the network learn more effectively.

Training the Network

Here’s where the magic happens. We’ll train our network to recognize patterns:

import torch.optim as optim

# Set device (GPU if available, otherwise CPU)
device = torch.device("cuda:0" if torch.cuda.is_available() else "cpu")
print(f"Using device: {device}")
# Initialize the network
net = SimpleCNN().to(device)
# Loss function and optimizer
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(net.parameters(), lr=0.001)
# Training loop
num_epochs = 10
for epoch in range(num_epochs):
    running_loss = 0.0
    
    for i, data in enumerate(trainloader, 0):
        # Get inputs and labels
        inputs, labels = data
        inputs, labels = inputs.to(device), labels.to(device)
        
        # Zero the parameter gradients
        optimizer.zero_grad()
        
        # Forward pass
        outputs = net(inputs)
        loss = criterion(outputs, labels)
        
        # Backward pass and optimize
        loss.backward()
        optimizer.step()
        
        # Print statistics
        running_loss += loss.item()
        if i % 200 == 199:
            print(f'[Epoch {epoch + 1}, Batch {i + 1}] loss: {running_loss / 200:.3f}')
            running_loss = 0.0
print('Finished Training')

Understanding the Training Process

The training loop follows a simple recipe:

  1. Forward Pass: Feed images through the network to get predictions
  2. Calculate Loss: Compare predictions with actual labels
  3. Backward Pass: Calculate how to adjust weights to reduce loss
  4. Update Weights: Apply the adjustments using our optimizer

This cycle repeats thousands of times, and gradually, the network learns to recognize patterns.

Evaluating the Model

Let’s see how well our network performs:

correct = 0
total = 0

# Don't compute gradients during evaluation
with torch.no_grad():
    for data in testloader:
        images, labels = data
        images, labels = images.to(device), labels.to(device)
        
        outputs = net(images)
        _, predicted = torch.max(outputs.data, 1)
        
        total += labels.size(0)
        correct += (predicted == labels).sum().item()
accuracy = 100 * correct / total
print(f'Accuracy on test set: {accuracy:.2f}%')
# Per-class accuracy
class_correct = [0] * 10
class_total = [0] * 10
with torch.no_grad():
    for data in testloader:
        images, labels = data
        images, labels = images.to(device), labels.to(device)
        
        outputs = net(images)
        _, predicted = torch.max(outputs, 1)
        
        for i in range(len(labels)):
            label = labels[i]
            class_correct[label] += (predicted[i] == label).item()
            class_total[label] += 1
for i in range(10):
    accuracy = 100 * class_correct[i] / class_total[i]
    print(f'Accuracy of {classes[i]:5s} : {accuracy:.2f}%')

With this simple architecture, you should achieve around 70–75% accuracy after 10 epochs, which is quite impressive considering we’re classifying tiny 32x32 pixel images into 10 categories!

Improving Your CNN

Once you have this basic CNN working, here are some ways to improve it:

Add More Layers: Deeper networks can learn more complex features. Try adding another convolutional layer or two.

Batch Normalization: Add nn.BatchNorm2d after convolutional layers to stabilize training and often improve accuracy.

Learning Rate Scheduling: Reduce the learning rate as training progresses using torch.optim.lr_scheduler.

Data Augmentation: Experiment with more aggressive augmentation techniques like color jittering or rotation.

Different Architectures: Once comfortable, explore famous architectures like ResNet or VGG that use skip connections and other advanced techniques.

Common Pitfalls and How to Avoid Them

Overfitting: If training accuracy is high but test accuracy is low, your model is memorizing rather than learning. Add more dropout or data augmentation.

Vanishing Gradients: In very deep networks, gradients can become too small to update weights effectively. Use batch normalization or residual connections.

Underfitting: If both training and test accuracy are low, your model might be too simple. Try adding more layers or neurons.

Memory Issues: If you run out of GPU memory, reduce your batch size or model size.
