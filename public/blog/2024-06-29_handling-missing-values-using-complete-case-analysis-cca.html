Handling Missing Values Using Complete Case Analysis (CCA)

Missing values in datasets can significantly affect the performance of machine learning models. One common method to handle missing data is Complete Case Analysis (CCA), also known as listwise deletion. This article will explain how CCA works, its assumptions, advantages, disadvantages, and when to use it. We will also illustrate CCA using a sample dataset.

Complete Case Analysis (CCA)

Definition: Complete Case Analysis involves discarding any row in the dataset that contains one or more missing values. This method ensures that only observations with complete information are analyzed.

Assumptions:

  • Missing Completely at Random (MCAR): The assumption is that the probability of missingness is the same for all observations. This means that the missing data is unrelated to any other variable in the dataset. To check if data is MCAR, one can compare the distribution of variables before and after applying CCA.

Advantages:

  • Easy to Implement: CCA is straightforward to apply, typically using the dropna function in pandas.
  • Preserves Variable Distribution: Since only rows with missing values are removed, the distribution of variables with no missing data remains unchanged.

Disadvantages:

  • Data Loss: CCA can result in a significant loss of data, especially if the dataset has many missing values.
  • Potential Bias: If the missing data is not random, removing rows can alter the distribution of the data and introduce bias.
  • Production Issues: In a production environment, models trained with CCA may not know how to handle new missing data.

When to Use CCA:

  1. Data Should Be MCAR: Ensure that missing values are completely random.
  2. Limited Missing Data: If more than 5% of the data is missing, CCA is generally not recommended.

Example Using a Sample Dataset

Let’s consider a dataset on data science job applications, data_science_job.csv, and apply CCA to handle missing values.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# Load the dataset
df = pd.read_csv('data_science_job.csv')

Analyzing Missing Values

# Calculate the percentage of missing values
missing_values = df.isnull().mean() * 100
print(missing_values)

From the above output, we see that some columns have more than 5% missing values, while others have less. We will focus on columns with less than 5% missing data for CCA.

# Select columns with less than 5% missing values
cols = [var for var in df.columns if df[var].isnull().mean() < 0.05 and df[var].isnull().mean() > 0]
print(cols)

Applying CCA

# Drop rows with missing values in selected columns
new_df = df[cols].dropna()
print(df.shape, new_df.shape)

We can see that after applying CCA, the number of rows has reduced, indicating data loss.

Conclusion

Complete Case Analysis (CCA) is a straightforward method for handling missing data by removing rows with missing values. While it is easy to implement and preserves the distribution of variables with no missing data, it can lead to significant data loss and potential bias if the missing data is not random. Therefore, CCA is recommended when the data is missing completely at random (MCAR) and the proportion of missing data is relatively small.
